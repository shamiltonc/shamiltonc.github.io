This is the narrative of my first enhancement of my RESTful API, as required by my final Computer Science class at SNHU: Computer Science Capstone.

A. Briefly describe the artifact. What is it? When was it created?

The artifact is a RESTful API service that I created a few months ago for the course CS-340 Client/Server Development. It was the course’s final project. It demonstrates administrative skills with a MongoDB database by implementing CRUD methods to retrieve and update data, and finally by turning each CRUD method into a RESTful service.

B. Justify the inclusion of the artifact in your ePortfolio. Why did you select this item? What specific components of the artifact showcase your skills and abilities in software development? How was the artifact improved?

I selected this item because it met all requirements of our final project, which makes it a solid demonstration of various aspects of software development and computer science for my ePortfolio. Because of the improvements I must make, this project also shows proficiency in code review and enhancement.

C. Did you meet the course objectives you planned to meet with this enhancement in Module One? Do you have any updates to your outcome-coverage plans?

I am in the process of meeting the course objectives I planned to make. For this milestone, I added proper comments, ensured logical flow of code, and established security. My only update on my coverage plans thus far are to include a narrative about Security in the final product.

I began to consider security during my code review, where I realized that I had hardly thought about security while initially developing the program. As I made my enhancements for the software engineering requirement, I realized that security is somewhat built in. Because I used Codio, the program is already secured. Codio themselves state that code on their website is secure and safe. The MongoDB client was replicated from the school, thus it is secure as well.

D. Reflect on the process of enhancing and/or modifying the artifact. What did you learn as you were creating it and improving it? What challenges did you face?

As I was making my initial improvements -comments, ensuring coding best practices, and security- I learned the value of adding comments early on. If I had, I would not have had to re-evaluate my code so extensively to make these enhancements. I also have realized the importance of considering security in the initial concept phase of a program. Luckily, I had Codio and SNHU when developing this RESTful API. However, next time I may not. Thus, I need to consider security as early on as possible.
